# Picture Size Converter 사용 가이드

Picture Size Converter는 이미지를 드래그 앤 드롭해서 원하는 해상도와 포맷(JPG/PNG/WebP)으로 변환한 뒤, 원하는 위치에 저장하는 데스크톱 앱입니다.

## 1. 준비 사항

- macOS 또는 Windows
- Node.js 18 이상 권장
- npm

설치 확인 예시:

```bash
node -v
npm -v
```

## 2. 설치 및 실행

프로젝트 루트에서 아래 순서로 실행합니다.

```bash
npm install
npm start
```

실행되면 앱 창이 열리고, 중앙에 이미지 드롭 영역이 표시됩니다.

## 3. 기본 사용 방법 (변환 1회)

1. 이미지 넣기
- 드롭 영역에 이미지를 끌어다 놓거나, 드롭 영역을 클릭해서 파일을 선택합니다.
- 파일이 선택되면 "선택된 파일" 영역에 파일 경로가 표시됩니다.

2. 크기 입력
- 너비(px), 높이(px)를 입력합니다.
- 두 값 모두 1 이상의 정수여야 합니다.
- 처음 파일을 선택할 때 너비/높이가 비어 있으면 원본 이미지 크기가 자동으로 채워집니다.

3. 출력 포맷 선택
- JPG / PNG / WebP 중 하나를 선택합니다.

4. 변환 시작
- "변환하고 저장하기" 버튼을 클릭합니다.

5. 저장 위치 선택
- 저장 대화상자에서 경로와 파일명을 정한 뒤 저장합니다.
- 기본 파일명은 원본이 photo.png이고 1200x800, JPG를 선택했다면 다음 형태로 제안됩니다.
  - photo_1200x800.jpg

6. 완료 확인
- 하단 상태 메시지에 완료 경로와 파일 크기(KB)가 표시됩니다.

## 4. 변환 동작 상세

- 리사이즈 방식: cover
- 의미: 입력한 너비/높이를 정확히 맞추기 위해 이미지가 비율 유지 상태로 확대/축소되며, 필요 시 일부 영역이 잘릴 수 있습니다.
- 포맷별 출력 설정
  - JPG: 품질 90 (mozjpeg 사용)
  - PNG: compression level 9
  - WebP: 품질 90

## 5. 자주 보는 메시지와 해결 방법

### 먼저 이미지를 넣어주세요.
- 원인: 파일이 선택되지 않은 상태에서 변환 버튼을 누른 경우
- 해결: 이미지를 먼저 드롭 또는 선택

### 너비와 높이는 1 이상의 숫자로 입력해주세요.
- 원인: 비어 있음, 0 이하, 정수가 아닌 값
- 해결: 예) 800, 600처럼 정수로 입력

### 저장이 취소되었습니다.
- 원인: 저장 대화상자에서 취소 선택
- 해결: 다시 변환 버튼 클릭 후 저장 진행

### 오류: 이미지 경로가 없습니다.
- 원인: 비정상 입력 파일 또는 파일 경로 접근 문제
- 해결:
  - 로컬 파일을 다시 선택
  - 파일이 존재하는지 확인
  - 외장 드라이브/네트워크 경로라면 접근 권한 확인

## 6. 성능/품질 팁

- 아주 큰 이미지는 변환 시간이 늘어날 수 있습니다.
- 최종 용량을 줄이고 싶다면
  - JPG/WebP 선택
  - 필요한 해상도보다 과도하게 크게 지정하지 않기
- 투명 배경이 필요하면 PNG를 사용하세요.

## 7. 개발/패키징 명령어

```bash
npm start           # 개발 실행
npm run dist:dir    # mac 폴더 형태 빌드
npm run dist        # mac zip 빌드
npm run dist:win:dir # Windows 폴더 형태 빌드
npm run dist:win    # Windows zip 빌드
```

## 8. macOS에서 앱이 열리지 않을 때

개인 개발용 빌드는 코드 서명이 없을 수 있어 보안 경고가 뜰 수 있습니다.

- Finder에서 앱 우클릭 -> 열기
- 또는 시스템 설정 -> 개인정보 보호 및 보안에서 실행 허용

팀 배포용이라면 Apple Developer ID 인증서로 코드 서명을 설정하는 것을 권장합니다.
